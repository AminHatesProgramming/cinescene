# Retrieval Benchmarks

`latest.json` is generated by:

```powershell
.\.venv\Scripts\python.exe .\scripts\retrieval_quality_benchmark.py
```

The suite contains plot-level, scene-level, and Persian scene queries. It reports:

- `Hit@1`: expected title is the first result;
- `Hit@5`: expected title appears in the first five results;
- `MRR@10`: reciprocal-rank quality within ten results;
- mean end-to-end query latency;
- per-query expected rank and top-three titles;
- active model, embedding dimension, device, and movie/scene vector counts.

This is a compact regression suite for the included catalog, not a claim of universal movie-retrieval accuracy. A larger academic evaluation should use held-out scene descriptions with human relevance labels.

## Latest Verified Result

`latest.json` was generated with BGE-large on CUDA against 7,615 movie/title vectors and 158 scene vectors:

```text
Hit@1:             0.90
Hit@5:             1.00
MRR@10:            0.95
Mean latency:      0.2372 s
Embedding width:   1,024
```

`reranked.json` records the optional cross-encoder ablation. It scored lower (`Hit@1=0.70`, `Hit@5=0.80`, `MRR@10=0.7333`) on this suite, particularly on Persian and scene-specific cases. CineScene therefore keeps cross-encoder reranking off by default instead of selecting it based on intuition alone.
